Why Kotlin ?
Motivation

Created by JetBrains

For their internal purposes.
... Java codebase, Scala compilation was too slow, Groovy too dynamic ...

They work with various languages on daily basis.
... should be able to design one ...

Great IDE support and tooling.
... IntellyJ IDEA ...

Mature

Introduced in 2011. Version 1.5 now.

No more breaking changes in API since 1.0
... 1.0 was released on February 2016 ...

Defined process for components stability.
... different evolution speed per component (Lang itself, stdlib, dokka etc.) ...

Kotlin Philosophy

"If to choose one word to describe Kotlin’s design, it would be pragmatism ... type system that prevents bugs ... abstraction mechanisms that facilitate code reuse ... interoperability with existing code and infrastructure is crucial ... elegance, though highly appreciated, is not the primary goal here, the primary goal is being useful ... we are proud of our tooling as well as the language"

Pragmatic

Designed to solve real-world problems.
Based on industry experience - Tooling creators + years of usage early versions by adopters.

Don't enforce any particular programming style or paradigm.
... can be used just like Java (OOP/imperative) in the beginning ...

Language features designed with tooling in mind.
... IDE find more concise constructs, fix the problems etc ...

Concise

“make you code smaller”
... less code means fewer bugs :) ...
... developer spent (much) more time reading existing code then writing new one ...

Code contains meaning.
... less boilerplate (getters/setters/constructors etc.) ...

No “free” operator overloading.
... you can always find what you want in documentation ...
... no cryptic punctuation sequences ( hello Scala :) ) ...

Safe

Preventing certain kinds of errors.
... more than Java ...

NullPointerException
... better ways to handle nullable data ...

ClassCastException
... better ways to handle casting ...

Interoperable

You can call methods, extend classes, implement interfaces, apply Annotations to Kotlin classes.

You can debug code between Kotlin and Java (mixed code-bases)

Kotlin can be called from Java as well.
... classes and methods can be called like regular Java)

Kotlin is using Java collection library
... just extending it with Extension functions ...

Programming guidelines

Kotlin conventions

File instead of: class/interface/enum/object - multiple classes or only top-level (semantically related) declarations. CamelCase naming.

Omit root package (org.example.kotlin) in pure Kotlin project. Follow Java structure in mixed ones.
